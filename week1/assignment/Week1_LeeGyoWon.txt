Quiz 1-1 short,int,long의 차이점

	- short : -32,768 ~ 32,767 까지의 범위를 갖는다. 2byte(16bit)크기를 가진다.
	- int : -2147483648 ~ 2147483647 까지의 범위를 가진다. 4byte(32bit)크기를 가진다.
	- long : -9223372036854775808 ~ 9223372036854775807 까지의 크기를 가진다 8byte(64bit)크기를 가진다.

	결정적인 차이점은 연산 할 수 있는 수의 범위의 크기가 달라진다는 점이다.

Quiz 1-2 float와 double의 차이점

	- float : 32bit 실수형
	- double : 64bit 실수형

	부동소수점을 표현하는 방식이며 둘의 차이점은 신뢰도에서 있다. double이 더 넓은 크기의 수를 표현함으로써 더 정확한 값들을 표현 할 수 있다. 하지만 메모리의 차이또한 많이 난다.

Quiz 1-3 char와 String의 차이점

	- char : 데이터 타입, 문자
	- String : char를 담고 있는 클래스타입, 문자열, char[]를 사용 및 표현하기 용이하게 하기 위해 만든 클래스

Quiz 2-1 캡슐화란

	- 데이터를 보호, 즉 정보를 은닉하기 위해 데이터의 접근을 제한하는 것을 의미 또는 관련된 함수나 데이터들을 하나로 묶어 객체로 만드는 것을 의미.
	- 멤버 접근 제어자

	  * private : 같은 클래스에서만 접근가능. 상속이 불가능함. 외부객체 접근불가
	  * protected : 같은 패키지 내에서 접근 가능. 상속을 받았다면 다른 패키지에서도 접근가능.
	  * public : 모든 객체에서 접근이 가능함.
	  * default : 접근제어자가 없는 형태로 같은 패키지에서만 접근가능.

Quiz 2-2 상속이란

	- 부모클래스로부터 멤버변수, 함수를 물려 받는 것을 의미한다. (유지보수가 편하고, 메모리 ▽, 코드를 여러번 작성할 필요성이 줄어듬)

	  * 슈퍼클래스 : 상속해주는 클래스로 부모클래스를 말함.
	  * 서브클래스 : 상속받은 클래스로 자식클래스를 말함. 
	     ※ private : 상속 X, protected : 상속 O

Quiz 2-3 다형성이란
	- 타입(부모클래스가 같음을 의미)은 같지만, 속성이나 성질이 다른 객체를 선언하는 것.
          하나의 형태에 여러 가지 데이터들을 처리할 수 있도록 하는 성질을 말함.
          메소드 overriding,loading과 같은 것을 예로 말할 수 있음.
         
          * overriding
		- 메소드 overriding은 슈퍼클래스의 메소드와 동일한 메소드를 재정의하는 것.
		  메소드 overriding의 목적은 슈퍼클래스에 구현된 메소드를 무시하고 서브클래스에서 새로운 기능을 가진 메소드를 재정의 하고자 하는것.
                  조건 : 메소드의 이름, 인자타입 및 개수, 리턴타입이 모두 동일하여야 함.
          * overloading
		- 메소드 overloading은 같은 클래스나 상속에서 동일한 이름을 가진 메소드를 중복으로 작성하는 것.
                  메소드 overloading의 목적은 이름이 같은 여러 메소드를 중복 정의하여 사용이 편리하게 함.
                  조건 : 메소드의 이름은 반드시 동일하고, 인자의 개수나 타입은 달라야 함.
